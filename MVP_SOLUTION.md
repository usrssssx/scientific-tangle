# MVP-решение

## Цель MVP

Доказать, что задача решается не только как RAG-поиск по документам, а как карта знаний: документы, эксперименты, процессы, материалы, оборудование, числовые условия, эксперты и выводы связываются в единую структуру.

## Сценарий MVP

1. Пользователь задаёт вопрос на русском или английском языке.
2. Система выделяет материалы, процессы, свойства, географию, период и числовые условия.
3. Поиск находит релевантные источники в корпусе.
4. Граф связывает документы с процессами, материалами, экспериментами и экспертами.
5. Числовой слой проверяет диапазоны и ограничения.
6. Ответ собирается в формате аналитической справки: источники, факты, эксперименты, эксперты, пробелы, расхождения, уровень уверенности.

## Компоненты MVP

### 1. Data ingestion

Поддерживаемые форматы: `.md`, `.txt`, `.pdf`, `.docx`, `.csv`, `.json`, `.zip`.

При загрузке создаётся запись `source`, документ режется на чанки, запускается извлечение сущностей и фактов.

### 2. Domain dictionaries

Словари хранят синонимы и термины:

- `электроэкстракция` / `electrowinning`;
- `ПВП` / `печь взвешенной плавки` / `flash smelting furnace`;
- `циркуляция католита` / `catholyte circulation`;
- `сухой остаток` / `TDS`;
- `МПГ` / `PGM`.

### 3. Extraction

MVP извлекает:

- сущности: Material, Process, Equipment, Property, Geography;
- числовые условия: `200-300 мг/л`, `≤1000 мг/дм³`, `0.15-0.30 м/с`, `18-65 руб/м3`;
- связи: документ описывает сущность, процесс использует материал, процесс использует оборудование, эксперимент валидирует процесс.

### 4. Storage

MVP использует SQLite:

- `sources` — источники;
- `documents` + `documents_fts` — чанки и полнотекстовый индекс;
- `entities` — сущности графа;
- `graph_edges` — связи;
- `facts` — проверяемые утверждения и числовые ограничения;
- `experiments` — каталог экспериментов;
- `experts` — эксперты и лаборатории;
- `audit_log` — аудит действий.

### 5. Search and synthesis

Пайплайн поиска:

```text
natural query
  -> query parser
  -> lexical + synonym retrieval
  -> entity/fact filtering
  -> experiment and expert lookup
  -> contradiction/gap detection
  -> structured answer
```

### 6. Security MVP

В MVP есть роли:

- `external_partner`;
- `researcher`;
- `analyst`;
- `manager`;
- `admin`.

Уровни доступа к источникам:

- `public`;
- `internal`;
- `confidential`;
- `secret`.

В production это нужно заменить на SSO/AD/LDAP, ABAC/RBAC, политику классификации документов и аудит в SIEM.

## Что демонстрируют тестовые данные

1. Обессоливание воды при сульфатах/хлоридах/Ca/Mg/Na 200-300 мг/л и целевом TDS ≤1000 мг/дм³.
2. Циркуляция католита при электроэкстракции никеля и диапазон скорости 0.15-0.30 м/с / 0.20-0.35 м/с.
3. Распределение Au, Ag и МПГ между штейном и шлаком за последние 5 лет.
4. Закачка шахтных вод в глубокие горизонты и технико-экономические показатели.
5. Пробел знаний по холодному климату, кучному выщелачиванию и никелевой руде.
